# 提案：新增大运与流年分维度解读

## Why

当前系统已经可以生成四柱命盘和结构化大运/流年序列，但距离“可直接阅读的命理报告”还差关键一步：每个大运、每个流年缺少按事业、感情、健康、财富拆开的解释层。

**背景**：
- 原始目标明确要求“每个运每一年”的详细分析
- 当前输出更偏向计算结果，尚不足以直接驱动最终报告模板

**当前状态**：`luckCycles` 只包含起运时间、大运干支和流年序列，没有解释性字段。

**期望状态**：每个大运和每个流年都附带结构化解读，至少覆盖 `overall / career / relationships / health / wealth` 五个维度。

## What Changes

- 为大运和流年新增结构化分析数据
- 增加流运与原局的关系判断逻辑
- 将流年十神、五行交互、冲合刑害等信号转译为可读摘要
- 更新 CLI 输出和测试

## Impact

### 受影响的规范
- `spec/specs/bazi-core/spec.md` - 将新增流运解读相关需求

### 受影响的代码
- `src/types.ts` - 扩展流运分析类型
- `src/analysis.ts` - 新增大运/流年分析逻辑
- `src/bazi.ts` - 把分析结果挂到输出结构
- `tests/bazi.test.ts` - 新增验证

### 用户影响
- 命盘输出可直接用于“每运每年”的解读页面或报告模板
- 减少后续长报告生成时的二次拼装成本

### API 变更
- 当前无 HTTP API
- CLI JSON 输出结构会扩展，但保持向后兼容

## 时间线评估

中等规模，属于在现有引擎之上增加解释层的增量迭代。

## 风险

- 流运解读天然带有流派差异，当前实现必须明确是“结构化启发式分析”，而非完整门派定论
- 文本过度武断会影响后续产品合规与用户预期，需要保持“趋势/信号/建议”式表述
