# 提案：新增四柱八字核心引擎

## Why

当前仓库还没有任何可运行代码，也没有四柱八字方向的结构化能力。

**背景**：
- 目标产品已经明确要先专精“四柱八字”，暂不同时铺开六爻、紫微等体系
- 后续的长报告、流年解读、RAG 知识库都依赖一个稳定的命盘计算与分析底座

**当前状态**：仓库为空，缺少 OpenSpec 结构、代码骨架、排盘内核、测试和 CLI。

**期望状态**：先交付一个可运行的 TypeScript MVP，能够把阳历/农历输入转成结构化命盘，并输出基础关系分析与大运/流年序列。

## What Changes

- 新增 OpenSpec 变更 `add-bazi-core-engine`
- 新增 TypeScript 项目骨架与 `lunar-javascript` 集成
- 实现四柱、藏干、十神、五行、合冲刑害、大运流年等结构化输出
- 新增 CLI 与自动化测试

## Impact

### 受影响的规范
- `spec/specs/bazi-core/spec.md` - 将在后续归档时并入首批四柱八字核心需求

### 受影响的代码
- `src/` - 四柱八字内核、关系分析、CLI
- `tests/` - 核心测试
- `package.json` / `tsconfig.json` - 项目基础设施

### 用户影响
- 可以直接通过 CLI 生成结构化八字命盘 JSON
- 为后续 API、报告模板和知识库接入建立稳定输入输出模型

### API 变更
- 当前阶段无 HTTP API
- 新增本地 CLI 接口

### 需要迁移
- [ ] 数据库迁移
- [ ] API 版本提升
- [ ] 用户沟通
- [x] 文档更新

## 时间线评估

中等规模，适合先完成 MVP，再继续推进年度分析细化与文本报告引擎。

## 风险

- `lunar-javascript` 提供的是历法/八字底层，不是完整命理推断，需要在本仓库补足规则层
- 五行强弱、调候、格局和细节断法仍有大量流派差异，当前实现先提供“结构化分析底座”，不冒充完整流派结论
